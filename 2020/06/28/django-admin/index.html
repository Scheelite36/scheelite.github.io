阅读量:

该文章是慕课网的相关课程的学习笔记

  • Django后台管理就是对项目数据库表的简单维护

  • 从ORM模型中读取元数据，简单配置即可使用

框架配置

步骤一：INSTALLED_APP配置应用及依赖

django.contrib.admin、django.contrib.auth、django.contrib.contenttypes、django.contrib.messages、django.contrib.sessions

步骤二：模板选项配置TEMPLATES中的OPTIONS

  • 添加上下文处理器：context_processors

    django.contrib.auth.context_processors.auth

    django.contrib.messages.context_processors.messages

步骤三：添加中间件MIDDLEWARE

django.contrib.auth.middleware.AuthenticationMiddleware

django.contrib.messages.middleware.MessagesMiddleware

步骤四：配置URL

访问后台管理：http://127.0.0.1:8000/admin/

可以修改用户的相关权限，也可以通过组的方式给一组用户添加特定权限。

settings中的LANGUAGE_CODE = 'zh-hans'可以更换语言为中文，同时更换时区为TIME_ZONE = ‘Asia/Shanghai’

ORM模型配置

步骤一：在模块中新建admin.py文件

步骤二：封装模型自定义的管理后台功能和选项

  • 每个模型创建一个ModelAdmin的子类

    ModelAdmin配置选项

    • filed/exclude——需要编辑/不需要编辑的字段列表
    • form——自定义表单
    • list_display——在列表中显示的字段
    • ordering——指定排序规则

步骤三：INSTALLED_APPS配置应用

  • 需要管理的ORM模型自动搜索添加至后台管理

    如accounts.apps.AccountsConfig

使用admin的一些优化配置

  • 查询速度优化

    减少每页的数据大小，比如分页；减少SQL查询的次数，外键关联等一次性查询。

    list_per_page 分页大小，默认每页100条

    list_select_related 外键关联查询优化

    paginator 分页器

  • 字段格式化字段优化

    场景：将用户名、手机号等脱敏处理

    在定义的admin类中添加自定义方法，然后再将名称传入list_display。同时还能指定方法

  • 快捷搜索优化

    list_filter

  • 实现用户搜索

aaps.py添加verbose_name可修改在管理系统的显示名称, 模型也可以通过verbose_name修改显示的名称

添加自定义方法

思考：以下场景如何解决？

  • 批量禁用用户
  • 批量下线商品

我们需要选择一批对象，然后执行某个操作。

第一步：在相关的类中 定义操作函数

第二步：命名界面显示的操作名称

aciton_name.short_description = '将选中的商品下线'

第三步：配置ModelAdmin

另外，另一种方法是单独建立一个定义函数，同时类中的actions引用该函数。这样该定义函数就可以被其他的类也同时使用。

举例，批量禁用/启用选中用户/商品：

在公用的包中新增公用的自定义函数模块admin_actions.py

然后在各个模块的admin.py中就可以直接通过actions引用了

这用后台管理中就添加了自定义的启用/禁用对象的方法。

自定义模型的配置

思考：如下场景如何实现？

  • 商品的UID是商品的唯一标识，创建了就不允许修改
  • 商品的销售价格怎么保证输入的是正整数
  • 商品的类型是否可以改为radio单选

限制字段修改的方法：

  1. 在ORM模型中给字段添加属性editable=False

  2. admin.py的类中加上exclude=['colum_name']

    使用前两种的字段在后台管理系统中都不可见了

  3. admin.py的类中加上readonly_fields = ['colum_name']，该字段名会在后台显示，但是不可编辑

自定义后台模型的表单验证和自定义后台模型的界面显示：

新建后台编辑表单，添加clean方法验证，同时在admin中引用该表单

表单对应的模型中修改__str__方法可以修改后台的表单项的显示名字
